# WaslMed Doctor Platform - Complete Implementation Guide

## Overview

This document provides comprehensive implementation guidelines for the WaslMed Doctor Platform - a complementary application to the main WaslMed patient platform. The Doctor Platform allows healthcare providers to scan patient QR codes, request access to medical records, view and edit patient data with permission, and manage their patient connections.

## Core Functionality

1. **Authentication System**
   - Doctor registration and login
   - Secure session management
   - Profile creation and management

2. **Patient Connection Flow**
   - Multiple methods to connect with patients:
     - QR code scanning (camera)
     - QR code image upload
     - Manual ID entry
   - Connection request management
   - Patient list management

3. **Medical Record Access**
   - View patient medical records
   - Edit permitted sections
   - Access history tracking
   - Edit history logging

4. **Dashboard & Analytics**
   - Patient overview
   - Recent activities
   - Statistics and insights

## QR Code System

### QR Code Format

The QR codes generated by the WaslMed patient platform contain a URI in this format:

```
waslmed://medical-record/60f5e4d27e2d8e2a8cde1234
```

Where `60f5e4d27e2d8e2a8cde1234` is the MongoDB `_id` of a specific Medical Record document.

### Important Notes on QR Codes:

1. The QR code contains the raw MongoDB ObjectId without any encoding or transformation.
2. The ID is a 24-character hexadecimal string.
3. This ID points to the Medical Record, not the User/Patient.
4. QR codes are generated with "H" (high) error correction level to facilitate reliable scanning.

### QR Code Processing Flow

1. **Scan/Input Processing**:
   - When scanned via camera: Extract the URI and parse the ID
   - When uploaded as image: Process the image, recognize the QR code, then extract ID
   - When manually entered: Validate the format (24-char hex string) and use directly

2. **ID Validation**:
   - Ensure the extracted ID follows MongoDB ObjectId format
   - Example validation: `/^[0-9a-fA-F]{24}$/.test(id)`

3. **Access Request**:
   - Once validated, send to server with doctor's credentials

## Authentication & Security

### Doctor Registration Process

1. Basic information collection (name, email, password)
2. Professional details (specialization, license number, etc.)
3. Profile completion (education, certifications, etc.)
4. Email verification
5. Optional: License verification (may require admin review)

### Authentication System

Use JWT (JSON Web Tokens) for authentication similar to the patient platform:

1. Doctor logs in with email and password
2. Server validates credentials and issues JWT token
3. Token is stored in secure HTTP-only cookie or local storage
4. Token is included in all API requests as authentication

### Access Token Generation

For additional security, the platform generates specific access tokens when accessing medical records:

```typescript
// Generate access token for doctor
function generateAccessToken() {
  const token = crypto.randomBytes(32).toString('hex');
  const expires = new Date();
  expires.setHours(expires.getHours() + 24); // 24-hour expiry
  
  return {
    token,
    expires,
    created: new Date()
  };
}
```

## Doctor-Patient Connection Flow

### Requesting Access

1. Doctor scans QR code, uploads QR image, or enters record ID
2. System validates the ID format
3. API call is made to request connection:

```typescript
const response = await fetch('/api/doctor/patients/connect', {
  method: 'POST',
  headers: {
    'Content-Type': 'application/json'
  },
  body: JSON.stringify({
    medicalRecordId: recordId,
    reason: reasonForAccess
  })
});
```

4. Server creates connection request in MedicalRecord.connectedDoctors with status "pending"
5. Server adds patient to Doctor.patients with status "pending"
6. Patient is notified of connection request

### After Patient Response

1. When patient approves or rejects the request, the status is updated to "approved" or "rejected"
2. If approved:
   - Doctor can now access patient's medical record
   - Patient appears in doctor's patient list
3. If rejected:
   - Connection remains in history but is marked as rejected
   - Doctor can see the rejected status

## Medical Record Access

### Viewing Records

1. Doctor selects patient from their approved list
2. System logs access in MedicalRecord.accessLog
3. Medical record data is fetched and displayed

### Editing Records

1. Doctor navigates to editable section
2. Makes changes and submits with optional note explaining the change
3. System:
   - Creates entry in MedicalRecord.editHistory tracking all changes
   - Updates the actual field values
   - Logs the edit access in MedicalRecord.accessLog

### Editable Sections

By default, doctors can edit the following sections:
- Medications
- Conditions (status updates only)
- Visit notes
- Laboratory tests
- Imaging results

## Implementation Guidelines

### QR Code Scanning Implementation

Use a library like `react-qr-reader` for web or a native camera library for mobile apps:

```jsx
import React, { useState } from 'react';
import { QrReader } from 'react-qr-reader';

function QRScanner() {
  const [result, setResult] = useState('');
  const [error, setError] = useState(null);
  
  const handleScan = (data) => {
    if (data) {
      const medicalRecordId = extractMedicalRecordId(data);
      if (medicalRecordId) {
        setResult(medicalRecordId);
        // Process the ID - initiate connection request
      } else {
        setError('Invalid QR code format');
      }
    }
  };
  
  const handleError = (err) => {
    console.error(err);
    setError('Error scanning QR code');
  };
  
  // Extract ID from QR code URI
  const extractMedicalRecordId = (uri) => {
    if (uri.startsWith('waslmed://medical-record/')) {
      const id = uri.replace('waslmed://medical-record/', '');
      // Validate ID format
      if (/^[0-9a-fA-F]{24}$/.test(id)) {
        return id;
      }
    }
    return null;
  };
  
  return (
    <div>
      <h2>Scan Patient QR Code</h2>
      <QrReader
        constraints={{ facingMode: 'environment' }}
        onResult={(result, error) => {
          if (result) {
            handleScan(result?.text);
          }
          if (error) {
            handleError(error);
          }
        }}
        style={{ width: '100%' }}
      />
      {error && <p className="error">{error}</p>}
      {result && <p>Medical Record ID: {result}</p>}
    </div>
  );
}
```

### QR Code Image Upload

Implement image upload with QR code extraction:

```jsx
import React, { useState } from 'react';
import QrScanner from 'qr-scanner';

function QRUploader() {
  const [result, setResult] = useState('');
  const [error, setError] = useState(null);
  
  const handleFileUpload = async (event) => {
    const file = event.target.files[0];
    
    if (!file) return;
    
    try {
      const result = await QrScanner.scanImage(file);
      const medicalRecordId = extractMedicalRecordId(result);
      
      if (medicalRecordId) {
        setResult(medicalRecordId);
        // Process the ID
      } else {
        setError('Invalid QR code format');
      }
    } catch (err) {
      console.error(err);
      setError('Could not scan QR code from image');
    }
  };
  
  // Extract ID function same as above
  
  return (
    <div>
      <h2>Upload QR Code Image</h2>
      <input type="file" accept="image/*" onChange={handleFileUpload} />
      {error && <p className="error">{error}</p>}
      {result && <p>Medical Record ID: {result}</p>}
    </div>
  );
}
```

## Connection Process Implementation

### Requesting Connection

```typescript
async function requestConnection(medicalRecordId: string, reason: string) {
  try {
    const response = await fetch('/api/doctor/patients/connect', {
      method: 'POST',
      headers: {
        'Content-Type': 'application/json',
        'Authorization': `Bearer ${token}` // JWT token
      },
      body: JSON.stringify({
        medicalRecordId,
        reason
      })
    });
    
    const data = await response.json();
    
    if (!response.ok) {
      throw new Error(data.error || 'Failed to request connection');
    }
    
    return {
      success: true,
      message: data.message,
      status: data.status
    };
  } catch (error) {
    console.error('Connection request error:', error);
    return {
      success: false,
      message: error.message
    };
  }
}
```

### Viewing Patient Medical Records

```typescript
async function getPatientMedicalRecord(medicalRecordId: string) {
  try {
    const response = await fetch(`/api/doctor/medical-records/${medicalRecordId}`, {

    }

    };
  } catch (error) {
    console.error('Error fetching medical record:', error);
    return {
      success: false,
      message: error.message
    };
  }
}
```

### Editing Medical Records

```typescript

    };
  }
}
```

## Error Handling & Edge Cases

1. **Invalid QR Codes**:
   - Implement validation for the expected format
   - Show clear error messages for invalid codes
   - Provide guidance for retry or alternative methods

2. **Connection Already Exists**:
   - Check if connection already exists before creating a new request
   - Handle various statuses (pending, approved, rejected)
   - Provide appropriate user feedback

3. **Unauthorized Access**:
   - Implement proper authentication checks on all endpoints
   - Handle expired sessions gracefully
   - Provide clear error messages for unauthorized actions

4. **Network Issues**:
   - Implement retry mechanisms for failed API calls
   - Cache important data for offline access if appropriate
   - Show user-friendly connection error messages

5. **Record Not Found**:
   - Handle cases where the medical record ID is valid but doesn't exist
   - Provide clear user feedback and next steps

## Security Best Practices

1. **Authentication**:
   - Implement proper JWT handling with refresh tokens
   - Use HTTPS for all API calls
   - Set secure and HTTP-only cookies

2. **Authorization**:
   - Check permissions on all endpoints
   - Validate patient-doctor relationships before granting access

3. **Data Protection**:

   - Follow HIPAA/GDPR compliance guidelines for medical data

4. **Session Management**:
   - Implement proper timeout policies
   - Allow users to view and terminate active sessions
   - Enforce strong password policies
