# WaslMed Scanner App - Database Guide

## Overview

This document provides a comprehensive guide to the WaslMed database structure, focusing on how IDs are used and how the scanner app should fetch data from the MongoDB database. This will help you "vibe code" the scanner app to correctly retrieve and display patient medical records.

## Database Structure

WaslMed uses MongoDB, a NoSQL document database. MongoDB generates unique IDs for each document as `ObjectId` values, which look like this: `60f5e4d27e2d8e2a8cde1234`.

## Understanding MongoDB IDs

### ObjectId Structure

MongoDB's `ObjectId` is a 12-byte value with the following structure:
- 4 bytes: timestamp (seconds since the Unix epoch)
- 5 bytes: random value
- 3 bytes: incrementing counter

Example: `60f5e4d27e2d8e2a8cde1234`

### Important ID Fields

1. **User ID**: 
   - Field: `_id` in the Users collection
   - Usage: Identifies a unique patient or healthcare provider
   
2. **Medical Record ID**: 
   - Field: `_id` in the Medical Records collection
   - Usage: **This is the ID encoded in the QR code**
   - Relationship: Each Medical Record has a `user` field containing the User ID it belongs to

3. **Doctor ID**:
   - Field: `_id` in the Doctors collection
   - Usage: Identifies a unique healthcare provider
   - Relationship: Doctors can connect to multiple patients' medical records

## QR Code Content

The QR code generated by the WaslMed web application contains a URI in this format:

```
waslmed://medical-record/60f5e4d27e2d8e2a8cde1234
```

Where `60f5e4d27e2d8e2a8cde1234` is the MongoDB `_id` of a specific Medical Record document.

### Important Note on QR Code Format

The QR code contains the **raw** MongoDB ObjectId without any encoding or transformation. This is important for the scanner app to correctly extract and use the ID. The ID will always be a 24-character hexadecimal string.

## Doctor Connection Process

1. When a doctor scans a patient's QR code, the app will call the `/api/medical-record/access` endpoint with the doctor's credentials and the medical record ID.

2. If the doctor is not yet connected to the patient, a connection request is automatically created with status "pending".

3. The patient will see this request in their "Healthcare Providers" dashboard section under "Connection Requests".

4. The patient can approve or reject the doctor's request.

5. Once approved, the doctor can view and/or edit the patient's medical record.

## Edit History and Access Logs

The Medical Record document now includes:

1. **accessLog**: Tracks when doctors view a patient's record
   - Records both authorized and unauthorized access attempts
   - Includes timestamp, doctor information, and access type

2. **editHistory**: Tracks changes made by doctors
   - Records the field changed, old and new values
   - Includes timestamp, doctor information, and edit notes

3. **connectedDoctors**: Tracks doctor connection status
   - Records pending, approved, and rejected connection requests
   - Includes timestamp and reason for connection

## Fetching Data in the Scanner App

### API Endpoints

The main endpoint you need to call to retrieve medical record data is:

```
POST /api/medical-record/access
```

This endpoint handles:
- Logging the doctor's access attempt
- Creating a connection request if needed
- Returning the patient's medical record if authorized

Example request:

```javascript
fetch('https://waslmed.com/api/medical-record/access', {
  method: 'POST',
  headers: {
    'Content-Type': 'application/json'
  },
  body: JSON.stringify({
    medicalRecordId: '60f5e4d27e2d8e2a8cde1234',
    doctorId: '60f5e4d27e2d8e2a8cde5678',
    doctorToken: 'valid-auth-token',
    accessType: 'view' // or 'edit'
  })
})
```

### Making Edits to Medical Records

To edit a patient's medical record, doctors use:

```
PATCH /api/medical-record/access
```

Example request:

```javascript
fetch('https://waslmed.com/api/medical-record/access', {
  method: 'PATCH',
  headers: {
    'Content-Type': 'application/json'
  },
  body: JSON.stringify({
    medicalRecordId: '60f5e4d27e2d8e2a8cde1234',
    doctorId: '60f5e4d27e2d8e2a8cde5678',
    doctorToken: 'valid-auth-token',
    updates: {
      'conditions.0.status': 'resolved',
      'medications.1.dosage': '10mg'
    },
    editNote: 'Updated medication dosage and condition status'
  })
})
```

### Response Structure

The API response for accessing a record will have this structure:

```json
{
  "success": true,
  "authorized": true,
  "message": "Access logged successfully",
  "medicalRecord": {
    "_id": "60f5e4d27e2d8e2a8cde1234",
    "personalInfo": {
      // Personal information fields
    },
    "allergies": [
      // Allergy records
    ],
    // ... other medical record sections
    "lastUpdated": "2023-06-12T14:32:10.123Z"
  }
}
```

If the doctor is not authorized, the response will look like:

```json
{
  "success": false,
  "authorized": false,
  "message": "Connection request sent to patient for approval",
  "connectionStatus": "pending"
}
```

## Step-by-Step Scanner App Flow

1. **Scan QR Code**: Use a QR code scanner library in your React Native app.

2. **Parse URI**: Extract the Medical Record ID from the URI.
   ```javascript
   function extractMedicalRecordId(uri) {
     // Check if it's a valid WaslMed medical record URI
     if (uri.startsWith('waslmed://medical-record/')) {
       // Extract the ID without any additional encoding/decoding
       return uri.replace('waslmed://medical-record/', '');
     }
     return null;
   }
   ```

3. **Validate ID Format**: Ensure the ID follows MongoDB ObjectId format (24-character hex string).
   ```javascript
   function isValidObjectId(id) {
     return /^[0-9a-fA-F]{24}$/.test(id);
   }
   ```

4. **Log Access & Fetch Record**: Make API call to log access and (if authorized) retrieve the medical record.
   ```javascript
   async function accessMedicalRecord(medicalRecordId, doctorId, doctorToken) {
     try {
       const response = await fetch('https://waslmed.com/api/medical-record/access', {
         method: 'POST',
         headers: {
           'Content-Type': 'application/json'
         },
         body: JSON.stringify({
           medicalRecordId,
           doctorId,
           doctorToken,
           accessType: 'view'
         })
       });
       
       const data = await response.json();
       
       if (!response.ok) {
         throw new Error(data.error || 'Failed to access medical record');
       }
       
       if (!data.authorized) {
         // Show connection request sent message
         return { 
           authorized: false, 
           message: data.message,
           connectionStatus: data.connectionStatus
         };
       }
       
       return { 
         authorized: true, 
         medicalRecord: data.medicalRecord 
       };
     } catch (error) {
       console.error('Error:', error);
       throw error;
     }
   }
   ```

5. **Display Medical Record or Connection Status**: Present the information in a user-friendly way.

## Example: Complete Scanner Function

```javascript
import { Alert } from 'react-native';

async function handleScannedQRCode(uri, doctorId, doctorToken, navigation) {
  // Step 1 & 2: Extract Medical Record ID
  const medicalRecordId = extractMedicalRecordId(uri);
  
  if (!medicalRecordId) {
    Alert.alert('Invalid QR Code', 'This QR code is not a valid WaslMed medical record.');
    return;
  }
  
  // Step 3: Validate ID format
  if (!isValidObjectId(medicalRecordId)) {
    Alert.alert('Invalid Medical Record ID', 'The QR code contains an invalid medical record ID format.');
    return;
  }
  
  try {
    // Step 4: Log access & fetch record
    const result = await accessMedicalRecord(medicalRecordId, doctorId, doctorToken);
    
    // Step 5: Handle the result
    if (result.authorized) {
      // Navigate to display screen with medical record data
      navigation.navigate('PatientRecord', { medicalRecord: result.medicalRecord });
    } else {
      // Show connection pending screen
      navigation.navigate('ConnectionRequest', { 
        medicalRecordId, 
        status: result.connectionStatus,
        message: result.message
      });
    }
  } catch (error) {
    Alert.alert('Error', error.message || 'Failed to access medical record');
  }
}
```

## Common Errors & Troubleshooting

1. **Invalid ID Format**: Ensure the QR code contains a valid 24-character hexadecimal MongoDB ID.
   - Error: `Invalid medical record ID` (HTTP 400)
   - Solution: Validate the ID format before making the API call

2. **Record Not Found**: The medical record ID is valid but doesn't exist in the database.
   - Error: `Medical record not found` (HTTP 404)
   - Solution: Display a user-friendly error message

3. **Authentication Failed**: The doctor's token is invalid or expired.
   - Error: `Invalid or expired token` (HTTP 401)
   - Solution: Prompt the doctor to log in again

4. **Authorization Failed**: The doctor is not connected to this patient.
   - Not an error: The system will create a pending connection request
   - Solution: Show the doctor that their connection request is pending approval

## QR Code Troubleshooting

If the scanner app is having trouble scanning QR codes:

1. **Verify QR Content Format**: The QR code should contain exactly `waslmed://medical-record/{24-character-hex-id}` without any additional encoding.

2. **Check QR Code Size and Contrast**: Ensure the QR code is sufficiently large and has good contrast.

3. **Error Correction Level**: The QR codes are generated with "H" (high) error correction level, which should help with scanning even if the code is partially obscured or damaged.

4. **Manual Entry Fallback**: Consider implementing a manual entry option for cases where scanning fails, allowing healthcare providers to type in the ID directly.

## Security Best Practices

1. **Never Store Medical Records Locally**: Always fetch from the server and don't cache sensitive data.

2. **Implement Session Timeouts**: Automatically log out inactive users after a set period.

3. **Use Secure HTTP**: Always use HTTPS for API calls.

4. **Verify Provider Credentials**: Ensure the provider is authenticated before allowing QR code scanning.

5. **Implement Rate Limiting**: Prevent abuse by limiting API calls.
